## Fixes

* **fix(platform): migrate ACME HTTP-01 to ingressClassName API**: ClusterIssuer previously referenced IngressClass "nginx", which does not exist on cozystack clusters (Ingress classes are named per-tenant). This caused certificate issuance to fail when Ingress resources relied on the legacy `acme.cert-manager.io/http01-ingress-class` annotation. The release switches both sides to cert-manager's modern API:
  - ClusterIssuer: `http01.ingress.ingressClassName` (parameterized from `_cluster.expose-ingress`, default `tenant-root`)
  - Ingress annotation: `http01-ingress-ingressclassname`

  This fixes certificate issuance across tenant clusters and prevents cert-manager validation errors triggered by mixing old/new fields. ([**@myasnikovdaniil**](https://github.com/myasnikovdaniil) in #2439, original commit 72efe285c)
