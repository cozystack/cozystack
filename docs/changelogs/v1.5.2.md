# v1.5.2 (2026-06-30)

A patch release that fixes upgrade-blocking storageClass drift in the Harbor chart, repairs the SeaweedFS S3 service routing, and ships `talm` v0.32.0 with first-class encrypted user values.

## Fixes

* **fix(harbor): preserve jobservice and trivy storageClass on upgrade**: The `cozy-harbor` wrapper never forwarded `storageClass` to the jobservice `jobLog` PVC, so the admission controller stamped the cluster default onto the live PVC. Because `storageClassName` is immutable, the rendered manifest (no class) and the live object (admission-defaulted class) diverged and caused upgrade failures on the `harbor-*-system` HelmRelease. The chart now reads the live PVC's `storageClassName` via `lookup` and preserves it; fresh installs fall back to the configured `storageClass`. The same drift was present in the Trivy `StatefulSet`'s `volumeClaimTemplate` and receives identical treatment ([**@myasnikovdaniil**](https://github.com/myasnikovdaniil) in #2930, backport #3147).

* **fix(seaweedfs): point S3 consumers at the reachable S3 service**: The `s3-service-name.patch` renamed the S3 Service to `seaweedfs-s3`, but the S3 ingress backend, the iceberg ingress backend, and the COSI provisioner in-cluster endpoint still resolved through `componentName` (`<fullname>-s3`). With the shipped release name `seaweedfs-system`, they targeted `seaweedfs-system-s3` â€” a Service that does not exist â€” causing ingress-nginx to return 503 on the external S3 endpoint. The COSI endpoint also lacked an explicit port (defaulting to 443 instead of the S3 port 8333). All consumers are now aligned with the actual Service name, and the fix is captured in a downstream patch and covered by helm-unittests rendered under the production release name ([**@lexfrei**](https://github.com/lexfrei) in #3119, backport #3127).

## Other repositories

### talm v0.32.0

* **[talm] feat: first-class encrypted user values, honored at template and apply**: Adds support for arbitrary encrypted user secret value files (`*.encrypted.yaml`) that are age-decrypted in memory via `talm.key` at both `talm template` and `talm apply` time â€” plaintext never touches disk. `talm apply` now also honors all `--values`/`--set*` flags and `templateOptions` declared in `Chart.yaml`, which were previously silently dropped at apply time. Secrets are kept out of committed `nodes/*.yaml` files and terminal output by default; `--show-secrets` / `--show-secrets-in-drift` reveal them when needed ([**@lexfrei**](https://github.com/lexfrei) in cozystack/talm#222).

## Documentation

* **[website] docs(platform): document publishing.certificates wildcard options**: Adds missing reference documentation for `publishing.certificates.wildcard` and related options in the platform package value table, covering opt-in shared wildcard certificate issuance on the default ingress-nginx path ([**@lexfrei**](https://github.com/lexfrei) in cozystack/website#588).

* **[website] fix(docs): correct stale version pins and prevent next.yaml drift**: The `v1.5.yaml` version-pin data file was a verbatim copy of `next.yaml` that was never updated, causing every version-pin on the v1.5 documentation pages to render `cozystack v1.3.0` / Talos `v1.12.6` instead of `v1.5.0` / `v1.13.0`. The file is now corrected and a guard is added to prevent future `next.yaml` drift ([**@myasnikovdaniil**](https://github.com/myasnikovdaniil) in cozystack/website#589).

* **[website] feat(blog): add Cozystack v1.5.0 release announcement**: Publishes a blog post announcing the Cozystack v1.5.0 release, covering all major highlights including Gateway API via Cilium, TLS for managed databases and messaging, and other improvements introduced in the minor release ([**@tym83**](https://github.com/tym83) in cozystack/website#592).
