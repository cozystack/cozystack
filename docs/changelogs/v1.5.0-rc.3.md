# v1.5.0-rc.3 (2026-06-15)

Cozystack v1.5.0-rc.3 is a targeted fix release for the third v1.5.0 release candidate, shipping one critical API server bug fix that prevented `kube-controller-manager` from starting on clusters with `ValidatingAdmissionPolicy` resources targeting Cozystack-managed apps. It also incorporates **talm v0.31.0**, which adds DRBD-oriented sysctl and etcd backend defaults to the Cozystack preset, improves CLI error reporting for lookup failures, excludes loop devices from the LVM global filter, and fixes a misleading error when `talm template` is invoked without a template file.

## Fixes

* **fix(api): publish open spec as `x-kubernetes-preserve-unknown-fields` instead of `additionalProperties: true`**: The `cozystack-api` server was publishing the free-form `.spec` of aggregated `apps.cozystack.io` resources (`Tenant`, `Application`, …) using the boolean form `additionalProperties: true`. The Kubernetes `ValidatingAdmissionPolicy` status type-checker in `kube-controller-manager` nil-dereferences when it recurses into that OpenAPI node, causing a cluster-wide `kube-controller-manager` `CrashLoopBackOff` that stalled installs and blocked reconciliation. Replaced with the semantically equivalent `x-kubernetes-preserve-unknown-fields: true`, which the type-checker handles safely. A regression test (`TestPatchedSpecDoesNotPanicVAPTypeChecker`) reproduces the exact panic through the KCM entry point and verifies the fix ([**@myasnikovdaniil**](https://github.com/myasnikovdaniil) in #2867).

## Dependencies

* **[talm] Release v0.31.0** (github.com/cozystack/talm): Adds always-on DRBD-oriented sysctls (`tcp_orphan_retries`, `tcp_fin_timeout`, `netdev_max_backlog`, `netdev_budget`, `netdev_budget_usecs`) and an opt-in TCP keepalive toggle (`tcpKeepaliveTuning`) plus a tunable etcd backend quota (`etcd.quotaBackendBytes`, default 8 GiB) to the Cozystack preset — resolving TCP-port exhaustion during DRBD reconnect storms ([**@IvanHunters**](https://github.com/IvanHunters) in cozystack/talm#131); fixes a misleading `connection refused` error when `talm template` is run without `--file`/`--template` by guarding the empty-TemplateFiles check before any network I/O or chart rendering ([**@lexfrei**](https://github.com/lexfrei) in cozystack/talm#217); emits per-file progress on stderr (not stdout) so `talm template ... > file.yaml` produces clean YAML output, enriches `lookup` failures with kind/namespace/endpoint context and a six-class taxonomy of hints, and adds bounded retry with exponential backoff for transient connectivity errors ([**@lexfrei**](https://github.com/lexfrei) in cozystack/talm#212); excludes `/dev/loop` devices from the LVM `global_filter` in generated Talos machine configs to prevent unintended VG activation from loop-mounted images ([**@kvaps**](https://github.com/kvaps) in cozystack/talm#215).
