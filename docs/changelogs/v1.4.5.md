# v1.4.5 (2026-06-30)

A patch release fixing three regressions in the CSI layer, tenant Kubernetes HelmRelease timeouts, and Harbor upgrade stability. All main-repository changes are backports of PRs already merged to `main`. This release also coincides with the `talm` v0.32.0 release, which brings first-class encrypted user values.

## Fixes

* **fix(csi): detach orphan hot-plug volumes when VMI outlives its VM**: During node reprovisioning in tenant Kubernetes clusters the VMI and its hot-plug pod can outlive the parent VM. Because the upstream kubevirt-csi-driver treated a missing VM as "detach succeeded", the hot-plug pod kept the infra storage device exclusively attached to the source host; subsequent attaches on any other node failed — on DRBD with `failed to set source device readwrite`. The fix re-checks VMI volume status after the upstream unpublish path returns and detaches from the VMI directly when the parent VM is absent. Includes a bump of kubevirt-csi-driver to latest upstream (which incorporates the same upstream fix, kubevirt/csi-driver#184) ([**@kvaps**](https://github.com/kvaps) in #2866, backport #2965).

* **fix(kubernetes): add spec.timeout to tenant CSI HelmRelease**: The `kubevirt-csi-node` HelmRelease in tenant Kubernetes clusters was the only tenant addon HelmRelease without `spec.timeout: 10m` (cilium, coredns, ingress-nginx, and others all set it). With the default 5-minute Helm timeout, a slow tenant control-plane bringup — Kamaji APIs flapping, VM image pulls — could push the DaemonSet past the deadline, triggering helm-controller remediation that uninstalled and re-installed the DaemonSet mid-flight; any volume mount in progress at that moment was wedged for the rest of the pod's lifetime. Adding the 10-minute timeout aligns the CSI HelmRelease with its neighbours without changing remediation semantics for genuinely broken installs ([**@myasnikovdaniil**](https://github.com/myasnikovdaniil) in #2727, backport #2929).

* **fix(harbor): preserve jobservice and trivy storageClass on upgrade**: The Harbor chart never forwarded `storageClass` to the jobservice `jobLog` PVC, so the live PVC was created without `storageClassName` and the default-StorageClass admission webhook stamped the cluster default onto it. Because `storageClassName` is immutable, the rendered manifest (no class) and live object (admission-defaulted class) diverged, surfacing as upgrade failures on the `harbor-*-system` HelmRelease. The fix reads the live PVC's current StorageClass via `lookup` and falls back to the configured value for fresh installs. The trivy StatefulSet's `volumeClaimTemplate` had the same latent drift and receives the same treatment, keyed off `data-<release>-trivy-0`. Adds helm-unittest regression coverage and wires a `make unit-tests` target for the Harbor package ([**@myasnikovdaniil**](https://github.com/myasnikovdaniil) in #2930, backport #3151).

## Features and Improvements

* **[talm] feat: first-class encrypted user values, honored at template and apply**: talm v0.32.0 adds support for `values-secret.encrypted.yaml` — age-encrypted Helm value files that are decrypted in memory at `talm template` and `talm apply` time via `talm.key`, keeping secrets out of committed node files and terminal output. `talm apply` now also honours `--values`/`--set*` flags and `templateOptions` from `Chart.yaml`, eliminating the silent value-drop that occurred when re-rendering at apply time. Drift previews and post-apply verification redact secret-bearing fields by default; use `--show-secrets-in-drift` to reveal them. `talm init --encrypt`/`--decrypt` covers the `values-secret.yaml` ↔ `values-secret.encrypted.yaml` round-trip and gitignores the plaintext automatically ([**@lexfrei**](https://github.com/lexfrei) in cozystack/talm#222).

## Documentation

* **[website] docs(platform): document publishing.certificates wildcard options**: Adds documentation for two platform publishing certificate options that were previously undocumented in the reference table. `publishing.certificates.wildcard` enables opt-in shared wildcard certificate issuance (`*.<root-host>`) via DNS-01 for system services, avoiding Let's Encrypt per-domain rate limits at scale. `publishing.certificates.wildcardSecretName` lets the operator supply an existing wildcard TLS Secret instead of minting per-host ACME certificates. Includes gating conditions, precedence rules, and hostname coverage caveats ([**@lexfrei**](https://github.com/lexfrei) in cozystack/website#588).

* **[website] feat(blog): add KubeVirt VM Disk and VM Instance how-to post**: Adds a practical how-to covering VM Disk (`VirtualMachineDisk`) and VM Instance (`VirtualMachineInstance`) lifecycle management in Cozystack using KubeVirt, including disk creation, cloning, and instance operations ([**@tym83**](https://github.com/tym83) in cozystack/website#586).
