# v1.5.3 (2026-07-06)

A patch release with two targeted fixes: restoring persistent EFI/TPM firmware state for Windows VMs on KubeVirt, and eliminating ~2 seconds of per-request S3 latency in SeaweedFS caused by PostgreSQL connection churn.

## Fixes

* **fix(kubevirt-instancetypes): restore persistent EFI/TPM state**: Reverts an earlier change (#3006) that dropped persistent EFI/TPM firmware state for the `windows.11` / `windows.2k22` / `windows.2k25` instance-type preferences (and their `.virtio` variants) to work around a live-migration concern. That concern rested on an outdated assumption — that an RWO Filesystem backend-storage PVC pins a VM to its node. KubeVirt has live-migrated RWO-Filesystem backend storage since v1.4 (kubevirt/kubevirt#12629): on migration it creates a fresh target-state PVC and copies the small state blob, so persistent-EFI/TPM VMs on the default replicated storage live-migrate fine. With persistence restored, in-guest Microsoft UEFI CA / Secure Boot enrollment for Windows VMs now survives reboots and live migrations ([**@kvaps**](https://github.com/kvaps) in #3154, backport #3212).

* **fix(seaweedfs): configure postgres2 connection pool for the filer**: The vendored SeaweedFS chart only shipped connection-pool settings for the `mysql` store, which Cozystack does not use; the `postgres2` store it does use had no pool settings, so the filer opened a fresh PostgreSQL connection (TCP + TLS + SCRAM) for every metadata lookup — roughly 2 seconds of added latency per S3 request with ~5 lookups per operation. Setting `WEED_POSTGRES2_CONNECTION_MAX_IDLE=20`, `WEED_POSTGRES2_CONNECTION_MAX_OPEN=40`, and `WEED_POSTGRES2_CONNECTION_MAX_LIFETIME_SECONDS=600` lets connections be reused. Measured on a production cluster (12 nodes, 2 filers): PUT of a 4 KiB object dropped from 10s to 0.28s (p50), and HEAD from 2.6s to 0.16s (p50) ([**@mattia-eleuteri**](https://github.com/mattia-eleuteri) in #2906, backport #3194).

## Documentation

* **[website] docs(gpu): drop manual KubeVirt patch step now that the platform auto-wires permittedHostDevices**: Simplifies the GPU passthrough guide by removing a now-unnecessary manual KubeVirt CR patch step, since the platform automatically wires `permittedHostDevices` for GPU-enabled clusters ([**@lexfrei**](https://github.com/lexfrei) in cozystack/website#556).
