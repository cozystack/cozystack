# v1.5.0-rc.2 (2026-06-10)

A documentation-focused release candidate update adding a detailed GPU operator passthrough host-preparation guide, two new blog posts on Cozystack capabilities, and the `ansible-cozystack` v1.4.3 companion release with containerd CDI and LVM improvements.

## Documentation

* **[gpu-operator] Document passthrough variant host-driver prerequisite**: The `default` (passthrough) variant of the GPU operator requires a clean host with no pre-installed NVIDIA driver. The new "Passthrough variant: host preparation" section in `docs/gpu-vgpu.md` covers the exact symptom (init log + sandbox-validator crashloop + `lspci` evidence), a safe step-by-step driver removal procedure (including `nvidia_peermem`, initramfs update, and reboot), post-reboot verification steps, and a manual label-cleanup step for the `nvidia.com/gpu.deploy.driver=pre-installed` node label that the operator never automatically resets ([**@lexfrei**](https://github.com/lexfrei) in #2767).

* **[website] Exclude loop devices from LVM global_filter documentation**: Adds `/dev/loop` to the documented LVM `global_filter` in the Talos install guides (v0, v1.4, and next, for both `talos-bootstrap` and `talosctl` methods), preventing the host from scanning loop-mounted images during installation ([**@kvaps**](https://github.com/kvaps) in cozystack/website#563).

* **[website] Add "Platform-Managed Backups in Cozystack" blog post**: Introduction post covering the Cozystack backup API — BackupJob, Plan, Backup, RestoreJob, and BackupClass — walking through the tenant workflow for one-shot and scheduled backups, in-place vs copy-to restore, and admin customization of BackupClasses ([**@tym83**](https://github.com/tym83) in cozystack/website#566).

* **[website] Add "Managed Kubernetes how-to" blog post**: New how-to post demonstrating managed Kubernetes cluster deployment via the Dashboard and kubectl, including kubeconfig retrieval and built-in addon configuration (Cilium CNI, Cluster Autoscaler, cert-manager, ingress-nginx) ([**@tym83**](https://github.com/tym83) in cozystack/website#565).

## Dependencies

* **[ansible-cozystack] Release v1.4.3** (github.com/cozystack/ansible-cozystack): Fixes KubeVirt CDI block-volume imports failing with `Permission denied` by enabling containerd's `device_ownership_from_security_context` in the k3s CRI drop-in config ([**@lexfrei**](https://github.com/lexfrei) in cozystack/ansible-cozystack#48); adds an LVM `global_filter` to exclude DRBD, device-mapper, zvol, and loop devices from host LVM scanning, preventing conflicts with LINSTOR-backed storage volumes ([**@kvaps**](https://github.com/kvaps) in cozystack/ansible-cozystack#49); adds a CI version-sync invariant that catches dependency version drift across authoritative files ([**@lexfrei**](https://github.com/lexfrei) in cozystack/ansible-cozystack#34).
