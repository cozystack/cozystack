# v1.4.6 (2026-07-07)

A patch release with a single reliability fix for the `linstor` package, backported from `main`.

## Fixes

* **fix(linstor): set controller resource requests and relax liveness probe**: Under node CPU contention, the `linstor-controller` JVM could not answer its `GET /health` liveness probe within the piraeus-operator's default timing (`timeoutSeconds: 1`, `failureThreshold: 3`), because the operator-generated controller pod had no CPU request to guarantee it CPU shares. The kubelet then liveness-killed the container (exit 137), the controller crash-looped, its Service lost ready endpoints, `linstor-csi` started returning `EPERM`, and RWX-Filesystem `DataVolume`s never bound — breaking tenant Kubernetes / NFS provisioning. The chart now sets `resources.requests` (cpu `250m`, memory `700Mi`, requests only, no limits) on the `linstor-controller` container and relaxes the liveness timing to `timeoutSeconds: 5` / `failureThreshold: 6`, so the controller keeps answering health checks under load instead of being killed and restarted ([**@lexfrei**](https://github.com/lexfrei) in #3043, backport #3216).

## Documentation

* **[website] Drop manual KubeVirt patch step for GPU passthrough now that permittedHostDevices is auto-wired**: Simplifies the GPU passthrough guide by removing the manual `permittedHostDevices` KubeVirt CR patch step, since the platform now wires this automatically. Users following the guide no longer need an extra manual patch to get GPU passthrough working ([**@lexfrei**](https://github.com/lexfrei) in cozystack/website#556).
