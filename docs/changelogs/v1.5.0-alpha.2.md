# v1.5.0-alpha.2

A small follow-up alpha that fixes a CI reliability issue in the release workflow introduced while cutting v1.4.1 and v1.4.2, along with three documentation updates on the website.

> **⚠️ Alpha Release Warning**: This is a pre-release version intended for testing and early adoption. Breaking changes may occur before the stable v1.5.0 release.

## Development, Testing, and CI/CD

* **ci(release): repair orphaned draft tag_name on retag**: When a release-cut PR is merged after the underlying git tag was deleted and re-created (e.g. the release branch was force-updated between the initial tag push and the merge), GitHub orphans the corresponding draft release — its `tag_name` becomes `untagged-<hash>` while `name` is preserved. The "Publish draft release" step would then fail with `Draft release for <tag> not found`, blocking the release. The release-finalize workflow now detects this case, repairs the `tag_name` via the GitHub API, and continues publishing — eliminating the need for a manual fix-up or UI-based workaround ([**@myasnikovdaniil**](https://github.com/myasnikovdaniil) in #2761).

## Documentation

* **[website] Add platform-managed backups introduction blog post**: New blog post introducing the platform-managed backups feature and how it improves data protection workflows on Cozystack ([**@tym83**](https://github.com/tym83) in cozystack/website#566).

* **[website] Add Managed Kubernetes how-to blog post**: New blog post walking through provisioning and managing a Kubernetes cluster as a managed service on Cozystack ([**@tym83**](https://github.com/tym83) in cozystack/website#565).

* **[website] Exclude loop devices from LVM global_filter**: Documents the recommended `global_filter` configuration to exclude loop devices from LVM scanning, preventing spurious warnings and device enumeration issues on nodes that use loop-backed storage ([**@kvaps**](https://github.com/kvaps) in cozystack/website#563).
